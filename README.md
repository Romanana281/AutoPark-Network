# АвтоПарк — Система управления автопарком

**Распределённое Windows Forms приложение** для учёта легковых и грузовых автомобилей с поддержкой сетевого обмена данными между несколькими экземплярами программы.

## О проекте

«АвтоПарк» — это многофункциональная система, позволяющая вести базу автомобилей, разделённых на два типа: **легковые** и **грузовые**. 

Особенностью приложения является **сетевой режим работы**: несколько пользователей (или экземпляров программы) могут запускать сервер и обмениваться данными об автомобилях в локальной сети с помощью механизма автоматического обнаружения серверов (UDP Discovery) и TCP-соединения.

## Основные возможности

### Главное окно
- Просмотр списка марок и моделей автомобилей
- Добавление новых марок
- Изменение типа автомобиля (Легковой ↔ Грузовой) с автоматической конвертацией
- Валидация вводимых данных
- Сохранение и загрузка всей базы в формате **XML**

### Окно детальной информации по марке
- Отображение расширенных характеристик:
  - **Легковые**: гос. номер, мультимедиа, количество подушек безопасности
  - **Грузовые**: гос. номер, количество колёс, объём кузова
- Редактирование данных
- Генерация тестовых данных с имитацией загрузки (ProgressBar)

### Сетевые возможности
- Запуск собственного TCP-сервера на случайном свободном порту
- Автоматическое обнаружение других серверов в локальной сети (UDP Broadcast)
- Запрос и получение данных от других участников сети
- Отправка своих данных на другие серверы
- Распределённая загрузка данных между клиентами

## Технологии

- **Язык**: C#
- **Фреймворк**: .NET Framework 4.7.2
- **UI**: Windows Forms
- **Сеть**: 
  - TCP (`TcpListener`, `TcpClient`)
  - UDP Broadcast (Discovery)
- **Данные**: `BindingList`, `BindingSource`, XML-сериализация (`XmlSerializer`)
- **Многопоточность**: `BackgroundWorker`, `async/await`, `Task`

## Структура проекта

- `ListCar.cs` — главное окно приложения
- `Data.cs` — окно с детальной информацией по марке
- `Server.cs` — логика TCP-сервера и UDP discovery
- `Client.cs` — клиентская часть (поиск серверов и обмен данными)
- `Loader.cs` — генерация тестовых данных и прогресс загрузки
- `Car.cs`, `Truck.cs`, `IAuto.cs` — модели данных
- `Program.cs` — точка входа

## Как запустить

1. Откройте решение в Visual Studio
2. Соберите проект (`Build`)
3. Запустите приложение (`F5`)
4. Для работы в сети:
   - Нажмите кнопку **«Включить сервер»**
   - Запустите другие копии приложения на том же компьютере или в локальной сети
   - При выборе марки данные будут подгружаться как локально, так и с других серверов
